# Magic Simulator - 魔法阵模拟器与2D渲染器合并项目

## 项目概述

本项目成功将两个独立的C++项目合并为一个完整的应用程序：

- **magic**：复数魔法阵模拟器（后端计算逻辑）
- **2d-renderer**：基于OpenGL的光线追踪2D渲染器（前端显示）

合并后的程序实现了完整的交互式魔法阵模拟与可视化系统，用户可以通过鼠标绘制图形，系统实时计算复数场并渲染显示结果。

## 项目目标

1. **核心目标**：一比一复刻两个子项目的逻辑，仅编写"胶水代码"连接二者
2. **功能目标**：
   - 鼠标输入 → 画布A（cv数组）留下痕迹
   - 魔法模拟根据画布A计算fi复数场
   - 画布B（cvs数组）将实体像素与fi场结合得到材质
   - 渲染器渲染画布B到屏幕，完成闭环交互
3. **技术目标**：使用MinGW编译，生成单一可执行文件

## 已完成工作

### 1. 项目结构与构建
- **项目结构**：`merge/src/`、`merge/include/`、`merge/build/`
- **构建系统**：配置CMakeLists.txt支持MinGW编译
- **依赖管理**：集成了OpenGL (GLEW/GLFW) 依赖库

### 2. 核心算法集成
- **魔法模拟算法**：完整移植magic项目的核心函数
  - `magic_conduct()`：信号传导算法
  - `magic_parsing()`：图案解析
  - `magic_reco_ln()`：对数节识别
  - `magic_reco_exp()`：指数节识别
- **颜色映射**：实现了magic项目的`c2col1()`算法
- **渲染管线**：集成2d-renderer的OpenGL渲染系统
  - GPU计算着色器（约15ms/帧）
  - CPU回退路径

### 3. 胶水代码实现
- **数据流整合**：
  - `cv[W][H]`：画布A，记录实体像素（0=空，1=实体）
  - `fi[W][H]`：复数场，由魔法模拟计算
  - `cvs_r/g/b/a/lr/lg/lb[W][H]`：画布B，渲染数据
- **主循环集成**：
  ```cpp
  while (!glfwWindowShouldClose(window)) {
      magic_update();           // 更新魔法模拟
      magic_mapToCvs();         // 映射到渲染数组
      upWI(); upOI(); sscr();   // 处理输入和渲染
      glfwPollEvents();
  }
  ```

### 4. BUG修复与功能完善
- **线段消失问题**：修复了绘制线段后像素消失的问题
  - 原因：`magic_mapToCvs()`中实体像素的alpha值被设为全局透明度(0.5)
  - 修复：实体像素的alpha设为1.0，使其完全不透明，避免进入光线追踪路径
- **配置格式更新**：根据用户要求更新了cfg.txt格式
  ```text
  128 128                    // 画布尺寸
  Microsoft YaHei           // 字体
  0x111111                  // 环境光颜色
  0xDEDEDE                  // 字体颜色
  0.5                       // 全局透明度
  ```
- **输入系统集成**：实现了鼠标左键绘制、右键擦除功能
- **太阳光照禁用**：根据配置要求设置`sunsc = 0.0f`

### 5. 设计约束实现
- **实体像素透明度**：全局可配置值（默认0.5）
- **发光颜色**：与像素颜色相同（基于复数场与颜色的映射关系）
- **空白区域透明**：实现了用户的设计意图
- **用户控制限制**：用户只能控制实体像素的存在与否，颜色、亮度和透明度由模拟结果决定

## 技术架构

### 数据流设计
```
用户输入 → 鼠标坐标 → 画布A(cv数组) → 魔法模拟 → 复数场(fi数组)
                                    ↓
渲染显示 ← 画布B(cvs数组) ← 颜色映射 ← 材质计算
```

### 核心组件
1. **输入处理模块**：`upWI()`函数处理鼠标输入，调用`line_cvs()`绘制/擦除
2. **魔法模拟模块**：`magic_update()`每帧更新复数场
3. **渲染映射模块**：`magic_mapToCvs()`将复数场转换为渲染数据
4. **GPU计算模块**：OpenGL计算着色器实现高效光线追踪
5. **显示模块**：OpenGL纹理渲染和相机控制

### 关键技术
- **C++17标准**：使用现代C++特性
- **OpenGL 4.3**：计算着色器支持GPU加速
- **MinGW编译**：Windows平台兼容
- **跨平台设计**：理论上支持Linux/MacOS

## 使用方法

### 基本操作
- **鼠标左键**：绘制实体像素（黑色线段）
- **鼠标右键**：擦除像素
- **方向键**：移动相机
- **Ctrl+上/下**：缩放视图
- **Ctrl+0**：重置相机
- **Ctrl+N**：重置画布

### 功能说明
1. **简单绘制**：绘制简单线段（不构成特定图案）显示为黑色
2. **魔法模拟**：绘制对数/指数节图案会触发复数场计算
3. **实时渲染**：每帧更新模拟结果并渲染显示
4. **透明效果**：实体像素alpha=1.0，空白区域alpha=0.0

## 设计哲学

### 魔法模拟的核心思想
- **fi数组基础值为0**：这是设计中有意为之的特性
- **静态像素不生成火花**：只有特定图案（对数/指数节）才会产生火花传导
- **颜色映射**：复数通过`c2col1()`算法映射到HSL颜色空间

### 渲染设计原则
- **实体像素不参与光线追踪**：alpha=1.0使其直接显示颜色
- **透明区域光线追踪**：空白区域进行完整的光线追踪计算
- **发光效果**：复数场的幅度决定发光强度
